### 🗺️ ROADMAP DE MIGRACIÓN: DEL VANILLA ARTESANAL AL REACT REACTIVO

No existe un comando mágico de Shell que traduzca "Vanilla a React" con 100% de precisión porque nuestra arquitectura es personalizada, pero yo puedo ejecutar una **refactorización masiva en bloque** que actúa como esa "traducción automática".

Aquí tienes el plan detallado en micro-pasos:

#### FASE 1: Reconfiguración del Núcleo (Entorno)
1.  **Instalación**: `npm install react react-dom @vitejs/plugin-react`.
2.  **Vite Sync**: Modificar `vite.config.js` para activar el plugin de React. Esto permite que el satélite entienda JSX.
3.  **Renombramiento**: Cambiar la extensión de los archivos UI de `.js` a `.jsx`.

#### FASE 2: Adaptación de la Materia (Score)
1.  **SovereignContext**: Crear un "Contexto de React" que envuelva la aplicación. Este contexto será el puente que conecte nuestro `AppState.js` y `NomonBridge.js` con los componentes de React.
2.  **Hook de Resonancia**: Crear un hook (`useIndraResonance`) que reemplace la lógica manual de carga de `app.js`. Este hook se encargará de "escuchar" al Bridge y avisar a React cuando los datos estén listos.

#### FASE 3: Refactor de Átomos (UI Components) - *La Traducción*
Este es el paso que pides. Traduciré cada clase manual a un componente funcional:
1.  **DataCard.jsx**: Eliminamos el `constructor` y el `render()`. El HTML que antes era un string dentro de un `template literal` se convierte en JSX puro.
2.  **Hero.jsx**: Lo mismo. Los eventos (on_click) pasan de ser `addEventListener` manuales a ser props de React (`onClick={...}`).
3.  **GridContainer.jsx**: Eliminamos el bucle `forEach` manual y usamos `.map()` de JavaScript, que es el estándar de oro en React.

#### FASE 4: Sustitución de Nervios (Router & Factory)
1.  **Eliminación de ComponentFactory.js**: React **ES** la factoría. No necesitamos una clase externa que decida qué renderizar; React lo hace nativamente.
2.  **SovereignRouter.jsx**: Crearemos un componente de ruteo simplificado que lea el `RouteMap` de NomonBridge y decida qué componentes mostrar basándose en el estado.

#### FASE 5: Ignición y Purga (app.jsx)
1.  **Punto de Entrada**: `app.js` se convierte en `main.jsx`. Usaremos `createRoot` para montar la aplicación.
2.  **Purga de Vanilla**: Eliminaremos físicamente todos los archivos de manipulación manual del DOM y las referencias a `appendChild`, `innerHTML` y `querySelector` innecesarios.
